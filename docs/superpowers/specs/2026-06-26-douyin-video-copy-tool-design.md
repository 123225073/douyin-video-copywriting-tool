# 抖音爆款视频文案复刻工具设计

## 1. 产品定位

这是一个本地运行的简单 HTML 工具，用于输入抖音链接，提取视频，识别视频中的口播、字幕或画面文字，并通过 CPA 模型改写成可复用文案。

第一版目标是“能稳定完成单条视频处理”，不做复杂平台化能力。

## 2. 第一版范围

必须包含：

- 输入抖音链接。
- 提取视频并保存到本地。
- 在线播放提取后的视频。
- 下载视频文件。
- 自动判断视频是否有可识别声音。
- 有声音时，提取音频并识别口播文案。
- 无声音或只有音效时，抽帧识别画面文字。
- 有字幕时，尽量识别字幕并和口播结果合并。
- 展示识别结果，并标明来源：语音、字幕、画面文字、AI 推测。
- 支持 CPA API Key 设置。
- 支持从 CPA 获取模型列表。
- 支持下拉选择默认改写模型。
- 支持一键 AI 改写整条文案。

第一版不包含：

- Coze 工作流兜底。
- 多供应商模型网关。
- 复杂用户权限。
- 批量处理。
- 自动发布抖音或其他平台。
- 云端部署。

## 3. 抖音视频提取

优先使用一个可实际跑通的开源解析方案，例如 `Douyin_TikTok_Download_API`。如果实际测试不可用，再考虑替换为 `yt-dlp`。

第一版只保留一个主解析链路，不增加 Coze 兜底，避免功能复杂。

解析失败时，页面直接显示明确原因，例如：

- 链接无效。
- 视频无法访问。
- 平台风控导致解析失败。
- 当前解析器暂不支持该链接。

## 4. 识别逻辑

### 4.1 有声音的视频

流程：

1. 从视频中提取音频。
2. 调用语音识别能力生成口播文案。
3. 同时抽取关键帧识别字幕。
4. 合并口播和字幕，输出完整文案。

输出时要说明哪些内容来自语音，哪些来自字幕。

### 4.2 无声音或只有音效的视频

流程：

1. 抽取视频关键帧。
2. 使用 OCR 识别画面中的文字。
3. 去重、排序、合并成文案。

如果画面里没有文字，工具不能假装读取到了原文，只能标记为“AI 根据画面推测”。

## 5. CPA 接入

第一版只做简单配置：

- 输入 CPA API Key。
- 点击获取模型列表。
- 下拉选择默认模型。
- 保存设置。

API Key 保存在本地后端配置文件中，前端只显示“已保存”，不明文回显。

AI 改写时使用用户保存的默认模型。

## 6. AI 改写

改写输入是识别整理后的完整文案。

第一版输出 3 个版本：

- 原文整理版：尽量还原原文逻辑。
- 爆款口播版：更适合短视频表达。
- 强转化版：更适合带货或引导行动。

改写结果支持复制。

## 7. 页面设计

页面使用简单三段式结构：

- 顶部：抖音链接输入、提取按钮、任务状态。
- 左侧：视频播放器、下载按钮。
- 右侧：识别文案、AI 改写区、CPA 设置区。

任务状态需要清楚显示：

- 等待输入。
- 正在解析视频。
- 正在下载视频。
- 正在识别语音。
- 正在识别画面文字。
- 正在 AI 改写。
- 完成。
- 失败。

页面不需要营销式首页，打开就是工具本体。

## 8. 后端设计

后端负责：

- 接收抖音链接。
- 调用视频解析器。
- 保存视频文件。
- 提供本地视频播放地址。
- 提供下载地址。
- 调用 FFmpeg 提取音频和关键帧。
- 调用语音识别。
- 调用 OCR。
- 调用 CPA 文案改写。
- 保存 CPA API Key 和默认模型。

前端不直接调用 CPA，也不直接保存 API Key。

## 9. 验收标准

第一版完成后需要验证：

- 输入真实抖音链接后，可以提取视频。
- 提取后的视频可以在线播放。
- 视频文件可以下载。
- 有口播的视频可以识别出语音文案。
- 无口播但有字幕的视频可以识别出画面文字。
- 无声音、无字幕、无画面文字的视频会明确提示只能 AI 推测。
- CPA API Key 可以保存。
- 模型列表可以获取并下拉选择。
- 选择模型后可以完成文案改写。
- 页面不会在处理过程中卡死。

## 10. 风险

- 抖音解析能力可能受平台风控影响，不能承诺所有链接都成功。
- 第一版不做 Coze 兜底，解析失败时直接提示失败。
- 无声音且画面无文字的视频无法读取原文，只能推测。
- 公开部署成下载站有合规风险，第一版定位为本地自用工具。
- 如果 CPA 不支持语音识别，需要先用本地语音识别方案。

## 11. 商业级视觉标准

页面目标不是普通工具页，而是可商业交付的高科技创作工作台。第一版即使功能简单，也必须让用户一打开就感觉这是一个专业、昂贵、可信的 AI 视频分析产品。

视觉关键词：

- 高科技。
- 炫酷但不花哨。
- 商业级。
- 专业可信。
- 操作高效。
- 视频内容分析感强。

设计要求：

- 首屏必须直接展示工具主体，不能做营销落地页。
- 页面要有明确的高科技视觉记忆点，例如视频分析波形、时间轴、识别进度、AI 文案生成流、发光状态线或数据面板。
- 不能做成普通表单页，也不能像廉价后台管理系统。
- 不使用大面积单一紫色、深蓝或纯黑，避免廉价模板感。
- 必须有清楚的信息层级：链接输入、视频预览、识别结果、AI 改写结果、CPA 设置各自边界明确。
- 任务处理中必须有明显动画反馈，包括解析中、识别中、改写中。
- 所有按钮、输入框、下拉框、播放器、结果区都要有统一风格。
- 桌面端优先，尺寸按 1440 x 1024 设计；后续再适配移动端。

## 12. 推荐视觉方向

后续视觉探索优先做 3 个方向，用户选择其中一个再进入开发：

### 方向 A：霓虹视频中控台

偏深色高科技，适合“爆款视频情报分析”的感觉。中心是视频播放器，左右是识别面板和 AI 改写面板，底部有视频时间轴、声纹和字幕识别轨道。

优点：

- 最炫酷。
- 一眼有 AI 分析工具感。
- 适合做商业演示。

风险：

- 深色界面如果控制不好，容易压抑。

### 方向 B：银白 AI 创作工作台

浅色高级 SaaS 风格，配合少量青绿、蓝色发光线条。整体像高端生产力工具，重点突出“干净、专业、长期可用”。

优点：

- 更适合长期办公使用。
- 商业交付感强。
- 可读性最好。

风险：

- 炫酷感比方向 A 弱一些。

### 方向 C：短视频作战舱

结合视频运营、爆款拆解、文案复刻的感觉，页面中有素材卡片、识别轨道、爆款改写结果和转化标签。视觉更偏“内容增长工具”。

优点：

- 最贴近抖音爆款复刻场景。
- 能体现业务价值。
- 适合后续扩展批量分析。

风险：

- 信息密度较高，需要控制排版。

## 13. 视觉验收标准

页面完成后，必须按以下标准验收：

- 1440 x 1024 桌面首屏无横向溢出。
- 390px 手机宽度不重叠、不截断关键文字。
- 页面打开后 3 秒内能看懂这是“抖音视频文案识别和改写工具”。
- 用户能一眼找到链接输入框和提取按钮。
- 视频预览、文案识别、AI 改写三块区域同时可见或有明确切换。
- 处理中有动效，不允许按钮点击后无反馈。
- 不能出现普通表格后台、廉价模板、信息堆叠、按钮风格不统一。
- 最终截图需要达到可对外演示、可商业报价、可作为 10W+ 项目初版交付的视觉质量。
